JavaScript의 난수는 Math에 들어있다. Math.random() 이게 난수를 가져오는 방법인데 Java와는 다르게 0 이상 1미만 중의 랜덤한 실수를 준다. (0.521845 뭐 이런식으로 말이다.)

그래서 보통 Math.random() * 30 (최대치로 지정할 수) 이렇게 max를 곱해서 이용하는데 소숫점 뒤에 붙어오는 실수들은 여전하다.

난수를 쓰는 보통의 경우 정수가 필요할텐데 이를 위한 기능 또한 Math에 포함 되어있다.

| Math.floor() | Math.ceil() | Math.round() |
| --- | --- | --- |
| 내림 | 올림 | 반올림 |

다만 내림을 하게될 경우 최대치로 지정한 수를 뽑을 수 없고 올림을 이용하게될 경우 0을 뽑을 수 없게 되니 만약 30을 max로 정한다 했을 때

```jsx
Math.floor(Math.random() * 31); //floor의 경우
Math.ceil(Math.random() * 31) - 1; //ceil의 경우
```

이렇게 해야 되는데 왜 영상에서 floor만 썼는지 알 것 같다.

길이가 짧아 좋은게 눈에 보이긴 하지만 이 랜덤숫자를 배열의 index로 이용한다고 생각하면 floor가 더욱 찰떡일 수 밖에 없다.

```jsx
const array = ["1", "2", "3", "4", "5"];
// 이렇게 글자 5개를 가진 array가 있다고 하자.
// 위의 글자를 랜덤하게 가져오고 싶다면

const randomArray = array[Math.floor(Math.random() * array.length)]
// 이렇게 범위 지정을 그 배열의 길이(5)로만 지정해주면
// 알아서 0 ~ 4 까지의 숫자만 나오기 때문이다.
```

근데 쓰고 보니 결국 -1 쓰고 안 쓰고 하나 차이이긴 하다.
