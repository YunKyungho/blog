먼저 리액트를 사용하기 위해 위 주소 두개를 html </body> 태그 밑 쪽에 먼저 import 해준 다.(최신 버전 따로 있음)

리액트를 import했기 때문에 createElement function을 가진 리액트 object에 접근이 가능하다.

(변수를 선언할 때는 변수명에 어떤 단어를 넣어도 상관 없으나 createElement(“span”) 처럼 ()안에는  반드시 생성하고자 하는 HTML 태그와 똑같아야한다.)

위에서 만든 객체를 보낼 때는

```jsx
ReactDOM.render();
```

ReactDOM.render() - render는 react element를 가지고 HTML로 만들어 배치해준다.

```html
<div id="root"></div>
```

```jsx
const root = document.getElementById("root");
const span = ReactDOM.render(생성한 객체 여기서는 span, 객체를 보낼 위치 여기서는 root);
```

위 같은 형식으로 쓰이는데

div같은 DOM객체를 하나 만들어 id를 부여한 뒤 Js에서 getElementById 등으로 그 id를 불러와 변수에 담아주고 뒤의 자리에 넣어준다.

createElement()를 사용할 때 객체의 종류 외에도 여러 값을 담아줄 수 있는데

```jsx
const span = React.createElement("span", {span의 property}, “span의 content”);
```

이런식으로 이용한다.

property 자리에 여러 속성을 넣고 싶을 때는 객체 형태로 값을 계속 넣어줄 수 있다.

그런데 굳이 span 하나 생성하는데 이렇게 많은 소스를 적을 필요가 있을까 그냥 HTML에 직접 <span></span>을 하면 되는데 말이다. 물론 이 생각이 맞다.

위 작업들은 단지 React JS가 우리가 해왔던 방식을 거꾸로 하고 있다는 것을 이해하기 위한 것이다.

JS는 HTML을 먼저 만들고 그걸 JS로 가져와서 HTML을 수정하는 형식이었다면.

리액트는 모든 것이 JS에서 시작하며 그 다음에 HTML이 되는 것이고 이것이 바로 React의 핵심이다.

우리는 Html을 만들고 찾아서 가져오고 그 다음 업데이트 하지 않고 React에게 업데이트 해야 하는 HTML을 업데이트 하라고 할 것이다.

만약 저 root div에 span 이외의 무언가를 같이 넣고 싶을 때는

```jsx
const button = React.createElement("button", {property}, “content”);
이렇게 버튼을 더 생성한 뒤

const container = React.createElement("div", {property}, [span, button]);
위에서 생성했던 버튼과 span을 배열 형태로 담은 div를 하나 더 만들고

ReactDOM.render(container, root);
렌더로 보내준다.
```

여기서 버튼에 어떤 이벤트를 감지하는 기능을 넣고 싶을 때는

```jsx
const button = React.createElement("button",
{
	onClick: () => console.log("im clicked")
}, “content”);

```

이렇게 property 자리에 onEvent: () ⇒ 작동 함수

이런 문법으로 작성 해주면 된다.

 

React는 interactive 한 어플에서 하는 작업들 모두가 event들을 감지하는 일이라는 것을 알고 있는 것이다.
