앞 글에서 기존에 사용했던 리렌더링 방식이다.

```html
	<script type="text/babel">
  const root = document.getElementById("root");

  function App(){
		const data = React.useState(0);
	  return( 
			<div>
	      <h3>누른 횟수: {counter}</h3>
	      <button onClick={countUp}>눌러라</button>
	    </div>
		);  
	}

  ReactDOM.render(<App />, root);
</script>
```

script 부분만 따로 보자.

우선 arrow function 형태를 그냥 함수로 바꿔주고 함수명도  App으로 바꿔줬다. 기존의 counter 작업을 하던 문구들도 전부 지워줬고 const data = React.useState(0); 라는 소스가 함수내에서 객체를 return 하기 전에 적혀있는데 이를 브라우저 요소 검사의 console에서 찍어보면 배열 형태로 값이 나온다.

[0, f] 이런식으로 말이다.

0은 내가 useState 뒤 괄호안에 넣어준 시작 값이고 f는 useState가 주는 함수다.

이 값을 좀 더 사용하기 편하게 받을 때 배열 형태로 받아줄 수가 있다.

```jsx
const [counter, modifier] = React.useState(0);
```

이러면 굳이 변수 하나로 받아서 data[0] 이런식으로 index로 지정할 필요가 없다. python의 tuple을 생각하면 될 것 같다.

여기서 useState로 받아온 modifier 함수는 값을 하나 받는데 어떤 값을 부여하던 그 값으로 업데이트하고 리렌더링을 일으키는 기능을 한다.

그래서

```jsx
	<script type="text/babel">
  const root = document.getElementById("root");

  function App(){
		const [counter, modifier] = React.useState(0);
		const onClick = () => {
			modifier(counter + 1);
		}
	  return( 
			<div>
	      <h3>누른 횟수: {counter}</h3>
	      <button onClick={onClick}>눌러라</button>
	    </div>
		);  
	}

  ReactDOM.render(<App />, root);
</script>
```

이렇게만 해주면 버튼을 눌렀을 때 onClick 함수가 불러지고 아까 counter의 값을 올리고 리랜더링 해줬던 작업을 modifier가 알아서 해주는 것이다.
