그리고 이용자가 인풋에 값을 입력하면 그 값으로 input의 value를 변경(리렌더링)해주어야 되기 때문에

```jsx
function App() {
	const [minutes, setMinutes] = React.useState();
	const onChange = (event) => {
		setMinutes(event.target.value);
	};
	return (
		<div>
			<h1>Converter</h1>
			<label htmlFor="minutes">Minutes</label>

			<input id="minutes" value={minutes} placeholder="Minutes" 
				type="number" onChange={onChange} />

			<label htmlFor="hours">Hours</label>
			<input id="hours" placeholder="Hours" type="number" />
		</div>
	);
}
```

이렇게 useState의 값을 배열로 받고 이용자가 minutes 인풋에 값을 입력하는 것을 감지하는 이벤트리스너 onChange를 통해 onChange 함수를 불러 발생한 이벤트에서 입력한 값을 토대로setMinutes(modifier)를 통해 리렌더링 하여 이용자가 보는 input의 값을 업데이트 시킨다.

(이거 근데 왜 이렇게 하는거지..? 그냥 input 만들면 되는거 아닌가 실시간으로 입력 값을 토대로 hours 값을 도출해내고 싶다면 이 방법이 맞는 것 같긴한데.. 흠..)

외부에서도 이 input의 value를 만질 수 있게 하기 위해서 하는거라고 한다.
