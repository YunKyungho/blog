# Blockchain Security: A Survey of Techniques and Research Directions

**IEEE Transactions on Services Computing, 2022 | 인용수: 198회**

## 연구 배경

블록체인은 "안전하고 공유 가능한 컴퓨팅"의 새로운 패러다임이다. 하지만 다양한 보안 이슈로 인해 발전과 확산이 심각하게 저해되고 있다.

기존 블록체인 보안 연구의 문제점:
- 대부분 **기술적 측면**에만 집중
- 비즈니스, 조직, 운영 이슈는 간과

이 논문은 **정보시스템 관점**에서 블록체인 보안을 체계적으로 정리했다.

## 블록체인의 4가지 핵심 요소

| 요소 | 역할 |
|------|------|
| **체인 구조** | 데이터 검증 및 저장 |
| **[[합의 알고리즘]]** | 데이터 생성 및 업데이트 |
| **[[암호화 기술]]** | 데이터 전송 및 접근 보안 |
| **[[스마트 컨트랙트]]** | 데이터 프로그래밍 및 연산 |

## PDI 모델: 블록체인 보안 프레임워크

저자들이 제안한 3계층 보안 모델:

```
┌─────────────────────────────────┐
│      Process Level (프로세스)      │  ← 비즈니스/운영
├─────────────────────────────────┤
│        Data Level (데이터)         │  ← 정보/거래
├─────────────────────────────────┤
│   Infrastructure Level (인프라)    │  ← 기술/네트워크
└─────────────────────────────────┘
```

### 1. 프로세스 레벨 (Process Level)

비즈니스 로직과 운영 관련 보안:

- **거버넌스 이슈** - 누가 블록체인을 관리하는가?
- **규정 준수** - 법적 요구사항 충족
- **감사 가능성** - 추적과 검증 가능 여부
- **조직 간 신뢰** - 컨소시엄 내 신뢰 관계

### 2. 데이터 레벨 (Data Level)

데이터 자체와 거래 관련 보안:

- **데이터 무결성** - 변조 방지
- **개인정보 보호** - 트랜잭션 프라이버시
- **접근 제어** - 누가 무엇을 볼 수 있는가
- **[[스마트 컨트랙트]] 취약점** - 코드 버그로 인한 손실

### 3. 인프라 레벨 (Infrastructure Level)

네트워크와 시스템 관련 보안:

- **[[합의 알고리즘]] 공격** - 51% 공격, 셀피시 마이닝
- **네트워크 공격** - [[이클립스 공격]], [[시빌 공격]]
- **[[암호화 기술]] 취약점** - 양자 컴퓨터 위협
- **노드 보안** - 키 관리, 지갑 보안

## 주요 보안 위협 정리

### 합의 관련 공격
| 공격 | 설명 |
|------|------|
| **51% 공격** | 해시파워 과반 확보로 체인 조작 |
| **셀피시 마이닝** | 블록을 숨겼다가 전략적 공개 |
| **롱 레인지 공격** | PoS에서 오래된 키로 대체 체인 생성 |

### 네트워크 공격
| 공격 | 설명 |
|------|------|
| **[[이클립스 공격]]** | 노드를 네트워크에서 고립시킴 |
| **[[시빌 공격]]** | 가짜 ID로 네트워크 장악 시도 |
| **DDoS** | 네트워크 마비 |

### 스마트 컨트랙트 취약점
- **재진입 공격** - The DAO 해킹 (2016, $60M 손실)
- **정수 오버플로우** - 계산 오류 악용
- **접근 제어 미흡** - 권한 없는 함수 호출

## 미래 연구 방향

저자들이 제시한 연구 과제:

### 비즈니스 관점
- 산업별 맞춤 보안 프레임워크
- 법적/규제적 요구사항 통합
- 조직 간 거버넌스 모델

### 기술 관점
- 양자 내성 암호화
- 프라이버시 보존 기술 (영지식 증명)
- 확장성과 보안의 균형

## 이 논문의 의의

1. **PDI 모델 제안** - 기술만이 아닌 비즈니스 관점 포함
2. **정보시스템 관점** - 실제 도입 시 고려사항 종합
3. **체계적 분류** - 흩어진 보안 이슈들을 3계층으로 정리

## 내 생각

기술적 보안만 다루는 논문은 많지만, "누가 운영하고, 어떻게 감사하고, 법적으로 어떻게 대응하나"까지 다룬 논문은 드물다. PDI 모델이 실용적인 이유.

특히 **프로세스 레벨** 보안은 실제 기업 도입 시 가장 큰 걸림돌이다. 기술은 되는데 거버넌스가 안 되면 아무 의미 없음.
