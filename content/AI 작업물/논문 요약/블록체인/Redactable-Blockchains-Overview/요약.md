## 한줄 요약

[[블록체인]]의 핵심 특성인 불변성을 유지하면서도 규제 준수나 오류 수정을 위해 데이터 수정이 가능한 [[수정 가능 블록체인]] 기술들을 종합 분석한 논문이다.

## 핵심 내용

### 불변성의 딜레마
- 블록체인 불변성: 데이터 무결성과 신뢰의 기반
- 하지만 현실에서는 수정이 필요한 상황 발생:
  - GDPR의 "잊힐 권리" 준수
  - 잘못 기록된 데이터 정정
  - 불법 콘텐츠 삭제
  - 민감 정보 제거

### 수정 가능 블록체인의 핵심 기술

#### 1. 카멜레온 해시 (Chameleon Hash)
- 트랩도어(비밀 키)를 가진 특수 해시 함수
- 트랩도어 소유자만 해시 충돌을 찾아 내용 변경 가능
- 외부에서는 수정 전후 블록이 유효해 보임
- Ateniese et al. (2017) 최초 제안

#### 2. 정책 기반 수정 (Policy-based Redaction)
- 사전 정의된 거버넌스 규칙에 따른 수정 허용
- 다수결, 다중서명, 위원회 승인 등
- 분산화와 수정 권한의 균형

#### 3. 투표 기반 수정
- 네트워크 참여자들의 합의로 수정 결정
- 민주적이나 느리고 복잡

### 설계 고려사항

| 속성 | 설명 |
|------|------|
| 투명성 | 수정 이력을 공개할 것인가? |
| 책임성 | 누가 수정 권한을 가지는가? |
| 분산화 | 단일 권한 없이 수정 가능한가? |
| 감사 가능성 | 수정 기록 추적 가능한가? |
| 안전성 | 악의적 수정 방지 가능한가? |

### 적용 분야
- **의료**: 환자 요청 시 기록 삭제
- **금융**: 규제 준수를 위한 데이터 관리
- **공급망**: 오류 데이터 정정
- **정부**: 민감 정보 보호

### 열린 문제
- 수정과 불변성의 적절한 균형
- 탈중앙화된 수정 권한 관리
- 수정 후에도 기존 참조 유효성 유지
- 성능 오버헤드 최소화

## 입문자를 위한 학습 포인트

1. **불변성이 항상 좋은가?**: 
   - 장부 조작 방지에는 좋음
   - 하지만 "실수로 전 여친 주소에 송금" 취소 불가
   - 불법 콘텐츠가 영구 저장되는 문제

2. **GDPR과의 충돌**: 
   - EU 법: 사용자가 자신의 데이터 삭제 요청 가능
   - 블록체인: 한 번 기록하면 삭제 불가
   - 어떻게 해결할 것인가?

3. **카멜레온 해시의 아이디어**:
   - 일반 해시: 내용 바뀌면 해시도 바뀜
   - 카멜레온 해시: 특별한 키 있으면 다른 내용도 같은 해시 생성 가능

4. **수정 ≠ 위조**:
   - 투명한 거버넌스와 감사 기록으로 신뢰 유지
   - "수정 가능"과 "마음대로 조작"은 다름

5. **실제 적용의 어려움**: 
   - 누가 수정 권한을 가질 것인가?
   - 남용 방지 방법은?
   - 탈중앙화와 어떻게 양립?
