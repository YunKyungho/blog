## 한줄 요약

보안운영센터(SOC)에서 대규모 언어 모델(LLM)을 활용하는 방법을 종합적으로 분석하고, 알림 처리, 위협 탐지, 인시던트 대응 자동화에서의 LLM 적용 현황과 과제를 제시한 서베이.

## 핵심 내용

### SOC의 현재 과제
- **경고 피로(Alert Fatigue)**: 대량의 보안 알림으로 인한 분석가 소진
- **인력 부족**: 숙련된 보안 전문가 확보 어려움
- **오탐(False Positive)**: 정상 활동의 잘못된 탐지
- **진화하는 위협**: 새로운 공격 기법에 대한 지속적 학습 필요
- **24/7 운영**: 지속적인 모니터링 요구

### LLM의 SOC 활용 분야
1. **알림 분류 및 우선순위화**
   - 대량의 알림 자동 분류
   - 심각도 기반 우선순위 지정
   - 관련 알림 클러스터링

2. **위협 인텔리전스**
   - 비정형 위협 보고서 분석
   - IoC(Indicator of Compromise) 추출
   - 위협 동향 요약

3. **인시던트 분석**
   - 로그 자연어 해석
   - 공격 체인 재구성
   - 근본 원인 분석

4. **대응 자동화**
   - 플레이북 생성 및 실행
   - 권장 조치 제안
   - 보고서 자동 작성

5. **지식 관리**
   - 보안 문서 검색 및 요약
   - Q&A 시스템
   - 신규 분석가 온보딩 지원

### LLM 적용의 도전과제
- **환각(Hallucination)**: 잘못된 정보 생성
- **도메인 특화 부족**: 보안 전문 지식 한계
- **지연시간**: 실시간 대응에 부적합할 수 있음
- **보안 우려**: LLM 자체의 취약점
