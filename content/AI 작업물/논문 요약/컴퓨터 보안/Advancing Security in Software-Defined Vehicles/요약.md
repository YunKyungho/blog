## 한줄 요약

소프트웨어 정의 차량(SDV)의 보안 위협과 방어 메커니즘을 체계적으로 분류하고, 기존 커넥티드 차량과 SDV를 구분하여 SDV 특화 보안 과제를 분석한 종합 서베이.

## 핵심 내용

### 소프트웨어 정의 차량(SDV) 정의
- 차량 기능이 소프트웨어에 의해 정의되고 업데이트되는 아키텍처
- OTA(Over-The-Air) 업데이트로 지속적 기능 개선
- 서드파티 애플리케이션 통합 가능
- 차량 수명 전반에 걸친 기능 확장

### SDV의 보안 위협
1. **OTA 업데이트 공격**: 악성 업데이트 주입, 롤백 공격
2. **앱 생태계 위협**: 악성 서드파티 앱, 권한 남용
3. **원격 접근 취약점**: 클라우드 연결을 통한 해킹
4. **CAN 버스 공격**: 내부 네트워크 메시지 조작
5. **데이터 프라이버시**: 운전 데이터 및 개인정보 유출

### 보안 분류체계(Taxonomy)
- **인증 및 접근 제어**: 디바이스/사용자 인증 메커니즘
- **암호화**: 통신 및 저장 데이터 보호
- **침입 탐지**: CAN 버스 이상 탐지
- **안전한 업데이트**: OTA 무결성 보장
- **격리**: 앱 및 ECU 격리

### SDV vs 일반 커넥티드 차량
| 특성 | 커넥티드 차량 | SDV |
|------|--------------|-----|
| 업데이트 | 제한적 OTA | 전면적 OTA |
| 앱 설치 | 불가 | 가능 |
| 아키텍처 | 하드웨어 중심 | 소프트웨어 중심 |
| 보안 위협 | 물리적 접근 위주 | 원격 공격 증가 |
