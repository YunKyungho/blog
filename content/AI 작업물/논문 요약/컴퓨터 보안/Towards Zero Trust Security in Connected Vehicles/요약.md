## 한줄 요약

제로 트러스트(Zero Trust) 보안 모델을 지능형 교통 시스템과 커넥티드 차량에 적용하는 방법을 종합적으로 분석하고, 차량 환경 특화 구현 전략을 제시한 서베이.

## 핵심 내용

### 제로 트러스트 원칙
- **절대 신뢰하지 않음**: 내부/외부 구분 없이 모든 요청 검증
- **지속적 검증**: 일회성 아닌 매번 인증
- **최소 권한**: 필요 최소한의 접근 권한만 부여
- **마이크로 세그멘테이션**: 네트워크 세분화로 측면 이동 방지
- **침해 가정**: 이미 침해당했다고 가정하고 설계

### 커넥티드 차량의 보안 과제
1. **다양한 통신 인터페이스**: V2X, 셀룰러, Wi-Fi, Bluetooth
2. **이기종 ECU 환경**: 다양한 벤더의 제어 장치
3. **실시간 요구사항**: 지연시간에 민감한 안전 기능
4. **제한된 리소스**: 임베디드 환경의 컴퓨팅 제약
5. **긴 수명주기**: 10-15년 이상의 차량 수명

### 제로 트러스트 구현 요소
- **신원 관리**: 디바이스, 사용자, 서비스 식별
- **다중 인증(MFA)**: 다요소 인증 적용
- **정책 엔진**: 접근 결정을 위한 정책 평가
- **정책 집행점(PEP)**: 실제 접근 제어 수행
- **지속적 모니터링**: 행동 분석 기반 이상 탐지

### 차량 환경 적용 시 고려사항
- 실시간 성능 요구사항 충족
- 제한된 컴퓨팅 자원에서 구현
- OTA 업데이트와의 통합
- V2X 통신에서의 신뢰 설정
- 안전-보안 균형

### 구현 사례
- ECU 간 인증 강화
- 동적 접근 제어 정책
- 이상 행동 탐지 및 대응
- 클라우드 연결 보안
