## 한줄 요약

딥러닝과 자연어처리(NLP)를 활용하여 CVE 설명에서 자동으로 CPE(Common Platform Enumeration)를 식별하고 취약점 요약을 어노테이션하는 CPE-Identifier 시스템 제안.

## 핵심 내용

### 연구 배경
- NVD에 등록되는 CVE 수가 급증
- 수동 CPE 매핑은 시간 소모적이고 오류 발생
- CVE 설명에서 영향받는 제품 자동 추출 필요

### CPE-Identifier 시스템

**파이프라인:**
1. **CVE 설명 전처리**
   - 텍스트 정제 및 토큰화
   - 도메인 특화 정규화

2. **엔티티 추출 (NER)**
   - 제품명, 벤더명, 버전 정보 추출
   - 보안 도메인 특화 NER 모델

3. **CPE 매핑**
   - 추출된 엔티티를 CPE 형식으로 변환
   - 기존 CPE 사전과 매칭

4. **요약 어노테이션**
   - 취약점 유형 분류
   - 영향도 정보 추가

### 딥러닝 모델
- BERT 기반 NER 모델
- 보안 텍스트로 파인튜닝
- BiLSTM-CRF 시퀀스 레이블링

### 평가 결과
- 높은 엔티티 추출 정확도
- 기존 규칙 기반 방법 대비 우수
- 다양한 CVE 형식에 일반화

### 한계점
- 복잡한 제품명의 추출 어려움
- 신규 벤더/제품 처리
- 버전 범위 해석 복잡성

## 입문자를 위한 학습 포인트

1. **NER(Named Entity Recognition)**: 텍스트에서 엔티티 추출
2. **CPE 형식**: cpe:2.3:a:vendor:product:version 구조
3. **BERT 기초**: 트랜스포머 기반 언어 모델
4. **보안 NLP**: 도메인 특화 텍스트 처리
