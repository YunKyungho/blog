## 한줄 요약

LLM을 활용하여 NVD의 취약점 레코드를 해당 취약점을 수정하는 Git 커밋(VFC, Vulnerability-Fixing Commit)에 자동으로 매핑하는 PatchSeeker 시스템을 제안.

## 핵심 내용

### 연구 배경
- NVD에 등록된 많은 CVE가 패치 커밋 링크 누락
- 수동으로 패치를 찾는 것은 시간 소모적
- 자동화된 VFC 식별의 필요성

### PatchSeeker 접근법
1. **LLM 기반 커밋 생성**
   - CVE 설명에서 예상 패치 커밋 내용 생성
   - 취약점 유형, 영향받는 코드 패턴 추론

2. **임베딩 기반 매칭**
   - 생성된 커밋과 실제 커밋의 의미적 유사도 계산
   - 코드 임베딩 모델 활용

3. **후보 순위화**
   - 유사도 점수 기반 커밋 후보 순위화
   - 상위 N개 결과 반환

### 기술적 세부사항
- 다양한 LLM 모델 비교 (GPT, CodeLlama 등)
- 코드 특화 임베딩 vs 일반 임베딩
- 커밋 메시지 + 코드 변경 모두 고려

### 실험 결과
- 기존 방법 대비 높은 정확도
- Top-5 추천에서 높은 재현율
- 다양한 언어 및 프로젝트에서 일반화

### 한계점
- LLM 환각으로 인한 오류 가능성
- 대규모 저장소에서 검색 비용
- 코드 변경 없는 설정 기반 수정 탐지 어려움
