## 한줄 요약

National Vulnerability Database(NVD) 사용자들의 태도, 경험, 기대, 어려움을 체계적으로 조사한 최초의 사용자 연구로, NVD의 실제 활용 현황과 개선 방향을 제시.

## 핵심 내용

### 연구 배경
- NVD는 전 세계에서 가장 널리 사용되는 취약점 데이터베이스
- 실제 사용자들의 경험과 요구사항에 대한 연구 부재
- NVD의 한계와 개선점 파악 필요

### 연구 방법
- 보안 전문가 대상 설문 조사 및 인터뷰
- 다양한 역할(개발자, 분석가, 관리자) 포함
- 정성적/정량적 분석 수행

### 주요 발견

**NVD 활용 현황:**
- 취약점 평가 및 우선순위화에 주로 사용
- 소프트웨어 구성 분석(SCA) 도구와 연계
- 규정 준수 및 보안 보고에 활용

**사용자가 느끼는 문제점:**
1. **처리 지연**: CVE 발표 후 NVD 분석까지 상당한 시간 소요
2. **불완전한 CPE**: 영향받는 제품 정보 누락 빈번
3. **일관성 부족**: 취약점 설명의 품질과 형식 편차
4. **패치 정보 부족**: 수정 커밋이나 패치 링크 누락
5. **CVSS 신뢰도**: 점수의 정확성에 대한 의문

**사용자 요구사항:**
- 더 빠른 CVE 분석 및 등록
- 완전한 CPE 매핑
- 패치/수정 정보 포함
- API 개선
- 커뮤니티 기여 허용

### 시사점
- NVD의 품질 향상 필요성 확인
- 자동화 도구 개발의 중요성
- 사용자 관점의 개선 우선순위 제시

## 주요 개념

- [[NVD]]: 미국 NIST가 관리하는 공식 취약점 데이터베이스
- [[CVE]]: 공개된 취약점의 고유 식별자
- [[CPE]]: 취약한 제품을 식별하는 표준
- [[CVSS]]: 취약점 심각도 점수

## 입문자를 위한 학습 포인트

1. **NVD 웹사이트 탐색**: nvd.nist.gov에서 실제 데이터 확인
2. **CVE 구조 이해**: CVE ID, 설명, 참조 링크 형식
3. **CVSS 계산기**: 점수 산정 방식 이해
4. **실무 활용**: SCA 도구와의 연계 방법
