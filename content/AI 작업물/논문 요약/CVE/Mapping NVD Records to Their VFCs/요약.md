## 한줄 요약

NVD 레코드를 취약점 수정 커밋(VFC)에 매핑하는 작업의 어려움을 체계적으로 분석하고, 매핑 품질과 자동화 도구의 한계를 실증적으로 연구.

## 핵심 내용

### 연구 목적
- NVD-VFC 매핑이 왜 어려운지 근본 원인 분석
- 기존 데이터셋의 품질 평가
- 자동화 도구의 성능 한계 파악

### NVD-VFC 매핑의 어려움

**데이터 품질 문제:**
1. **참조 링크 부재**: 많은 CVE에 수정 커밋 링크 없음
2. **부정확한 참조**: 잘못된 커밋을 가리키는 경우
3. **다중 커밋**: 하나의 CVE가 여러 커밋으로 수정
4. **분산 수정**: 여러 브랜치/포크에 걸친 수정

**기술적 어려움:**
- 커밋 메시지에 CVE 언급 없음
- 코드 변경과 취약점 설명의 불일치
- 리팩토링과 보안 수정 구분 어려움
- 대규모 프로젝트에서의 검색 복잡도

### 기존 데이터셋 분석
- 인기 있는 CVE-커밋 데이터셋들의 오류율 측정
- 수동 검증 결과 상당수 매핑이 부정확
- 데이터셋 품질이 후속 연구에 영향

### 자동화 도구 평가
- 기존 VFC 탐지 도구들의 성능 비교
- 정밀도와 재현율 트레이드오프
- 완전 자동화의 현실적 한계

### 제안 사항
- 매핑 검증을 위한 체크리스트
- 데이터셋 품질 평가 기준
- 수동 검증의 중요성 강조
