## 한줄 요약
연합 학습의 집계 기법을 다층 분류 체계로 정리하고, 이질성, 효율성, 보안, 프라이버시 관련 도전과 해결책을 종합적으로 분석한 서베이.

## 핵심 내용
[[연합 학습]]은 데이터를 중앙에 모으지 않고 분산된 클라이언트에서 모델을 학습하여 프라이버시를 보호하는 혁신적 패러다임이다. 데이터 프라이버시 우려와 데이터 사일로 문제가 증가하면서 전통적 중앙집중식 ML의 대안으로 부상했다.

### 연합 학습 핵심 과정
1. 중앙 서버가 초기 모델 배포
2. 각 클라이언트가 로컬 데이터로 학습
3. 모델 파라미터만 서버로 전송
4. 서버에서 파라미터 집계(Aggregation)
5. 업데이트된 글로벌 모델 재배포

### 집계 기법 분류
1. **아키텍처별**
   - 중앙집중식 (서버-클라이언트)
   - 분산형 (P2P)
   - 계층적 (다층 구조)

2. **동기화 방식**
   - 동기식: 모든 클라이언트 대기
   - 비동기식: 먼저 도착한 것부터 처리

3. **목표별**
   - 단일 모델 학습
   - 개인화 모델
   - 멀티태스크 학습

### 핵심 도전 과제
1. **데이터 이질성(Non-IID)**: 클라이언트별 데이터 분포 차이
2. **시스템 이질성**: 디바이스 성능 격차
3. **통신 효율성**: 대역폭 제한
4. **보안/프라이버시**: 모델 역공학, 멤버십 추론 공격

### 보안 강화 기법
- [[차분 프라이버시]]: 그래디언트에 노이즈 추가
- [[동형 암호화]]: 암호화된 상태로 집계
- 안전 다자간 계산(MPC)

## 주요 개념

- [[동형 암호화]]: 암호문 연산
- [[딥러닝]]: 모델 학습의 기반

## 입문자를 위한 학습 포인트
1. **데이터 안 모아도 학습 가능**: 프라이버시와 성능의 양립
2. **모델만 공유**: 원본 데이터는 디바이스에 유지
3. **집계가 핵심**: 어떻게 합치느냐가 성능 좌우
4. **완벽하진 않다**: 모델에서도 정보 유출 가능
5. **실제 활용 중**: 스마트폰 키보드, 의료 등에 적용
