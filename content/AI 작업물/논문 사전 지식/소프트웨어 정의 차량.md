## 정의

소프트웨어 정의 차량(Software-Defined Vehicle, SDV)은 차량의 주요 기능과 특성이 하드웨어보다 소프트웨어에 의해 결정되는 차세대 자동차 아키텍처이다. 무선 업데이트(OTA), 서드파티 앱 통합, 지속적인 기능 개선이 가능하다.

## 상세 설명

### 핵심 특징

1. **OTA 업데이트**: 딜러 방문 없이 무선으로 소프트웨어 업데이트
2. **앱 생태계**: 스마트폰처럼 외부 애플리케이션 설치 가능
3. **하드웨어 추상화**: 소프트웨어가 하드웨어 독립적으로 동작
4. **차량 수명 연장**: 소프트웨어 업데이트로 지속적 기능 향상

### SDV와 기존 차량의 차이

| 구분 | 기존 차량 | SDV |
|------|-----------|-----|
| 기능 업데이트 | 딜러 방문 필요 | OTA로 원격 업데이트 |
| 앱 설치 | 불가능 | 가능 |
| 개인화 | 제한적 | 소프트웨어로 확장 가능 |

### 보안 위협

- **OTA 업데이트 위협**: 악성 업데이트 주입
- **앱 보안**: 서드파티 앱을 통한 공격
- **원격 접근**: 해킹을 통한 차량 제어
- **데이터 프라이버시**: 운전 데이터 유출

### CAN 버스 보안

차량 내부 네트워크인 CAN(Controller Area Network)은 인증 메커니즘이 취약하여 주요 공격 대상이 됨.
