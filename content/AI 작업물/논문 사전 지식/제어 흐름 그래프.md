## 정의

**제어 흐름 그래프(Control Flow Graph, CFG)**는 프로그램의 실행 경로를 방향 그래프로 표현한 것이다. 각 노드는 기본 블록(basic block)을, 간선은 가능한 제어 흐름을 나타낸다.

## 핵심 개념

### 구성 요소
- **기본 블록**: 단일 진입/단일 종료의 연속된 명령어들
- **간선**: 조건문, 반복문 등에 의한 분기
- **진입점**: 함수/프로그램 시작점
- **종료점**: 함수/프로그램 종료점

### 활용 분야
- **컴파일러 최적화**: 불필요한 코드 제거, 루프 최적화
- **정적 분석**: 도달 가능성 분석, 데드 코드 탐지
- **취약점 탐지**: 위험한 실행 경로 분석
- **[[딥러닝]] 기반 분석**: [[그래프 신경망]]으로 CFG 학습

### 예시
```
[진입] → [조건 검사] → [참 분기] → [합류점] → [종료]
              ↓
         [거짓 분기] ────────↗
```
