# TEE (신뢰 실행 환경)

## 정의
TEE(Trusted Execution Environment)는 메인 운영체제와 격리된 안전한 영역에서 코드를 실행하고 데이터를 보호하는 하드웨어 기반 보안 기술이다.

## 핵심 특징
1. **격리(Isolation)**: 외부 프로세스나 OS로부터 보호된 실행 환경
2. **무결성(Integrity)**: 코드와 데이터가 변조되지 않았음을 보장
3. **기밀성(Confidentiality)**: 민감한 데이터가 외부로 유출되지 않음
4. **원격 증명(Remote Attestation)**: TEE가 정상 동작 중임을 외부에서 검증 가능

## 주요 구현체
- **Intel SGX (Software Guard Extensions)**: 가장 널리 사용되는 TEE
- **ARM TrustZone**: 모바일 기기에서 주로 사용
- **AMD SEV**: 가상 머신 수준의 메모리 암호화

## 블록체인에서의 활용
- **믹싱 서비스**: Obscuro 등에서 트랜잭션 순서 결정 시 사용
- **프라이버시 블록체인**: Oasis, Secret, Phala 등에서 기밀 스마트 컨트랙트 실행
- **MEV 완화**: 암호화된 멤풀에서 트랜잭션 순서 비공개 처리

## 장점과 한계
### 장점
- 하드웨어 수준 보안으로 높은 신뢰성
- 복잡한 암호화 없이 기밀 계산 가능

### 한계
- 하드웨어 의존성 (특정 제조사에 종속)
- 사이드채널 공격 취약점 (Spectre, Meltdown 등)
- 하드웨어 비용
