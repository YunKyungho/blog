## 정의

메모리 안전성(Memory Safety)은 프로그램이 메모리를 올바르게 접근하고 관리하여, 버퍼 오버플로우, use-after-free, 널 포인터 역참조 등의 메모리 관련 취약점이 발생하지 않는 상태를 의미한다.

## 상세 설명

### 주요 메모리 취약점 유형

1. **버퍼 오버플로우**: 할당된 메모리 영역을 초과하여 접근
2. **Use-After-Free**: 해제된 메모리를 다시 사용
3. **Double Free**: 같은 메모리를 두 번 해제
4. **널 포인터 역참조**: NULL 포인터를 통한 메모리 접근
5. **메모리 누수**: 사용이 끝난 메모리를 해제하지 않음
6. **정수 오버플로우**: 정수 연산 결과가 범위 초과

### 메모리 안전하지 않은 언어

C, C++ 등 수동 메모리 관리 언어에서 메모리 취약점 발생률이 높음:
- Microsoft: 보안 취약점의 70%가 메모리 안전성 문제
- Google Chrome: 취약점의 70%가 메모리 관련

### 메모리 안전 언어

- **Rust**: 소유권 시스템으로 컴파일 타임에 메모리 안전성 보장
- **Go**: 가비지 컬렉션과 배열 경계 검사
- **Java/C#**: 관리형 런타임으로 대부분의 메모리 오류 방지

### Rust의 접근법

- **소유권(Ownership)**: 각 값은 하나의 소유자만 가짐
- **빌림(Borrowing)**: 참조를 통한 임시 접근
- **수명(Lifetime)**: 참조의 유효 범위 추적
- **unsafe 블록**: 명시적으로 안전하지 않은 코드 격리

### CVE와 메모리 안전성

많은 CVE가 메모리 안전성 문제에서 기인:
- 원격 코드 실행
- 서비스 거부
- 권한 상승
- 정보 노출
