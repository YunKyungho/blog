## 정의

CVSS(Common Vulnerability Scoring System)는 소프트웨어 취약점의 심각도를 0.0에서 10.0 사이의 점수로 정량화하는 국제 표준 평가 체계이다. FIRST(Forum of Incident Response and Security Teams)에서 관리한다.

## 상세 설명

### CVSS 버전

- **CVSS v2**: 2007년 발표, 기본 점수 중심
- **CVSS v3.x**: 2015년 발표, 더 세분화된 평가 기준
- **CVSS v4.0**: 2023년 발표, 맥락 정보 강화

### CVSS 구성 요소

**1. 기본 점수 (Base Score)**
- 공격 벡터 (네트워크, 인접, 로컬, 물리적)
- 공격 복잡도 (낮음, 높음)
- 필요 권한 (없음, 낮음, 높음)
- 사용자 상호작용 (필요 없음, 필요)
- 범위 (변경됨, 변경 안됨)
- 영향도 (기밀성, 무결성, 가용성)

**2. 임시 점수 (Temporal Score)**
- 익스플로잇 코드 성숙도
- 해결책 수준
- 보고서 신뢰도

**3. 환경 점수 (Environmental Score)**
- 조직별 맞춤 영향도
- 수정된 기본 점수

### 점수 등급

| 점수 범위 | 심각도 |
|-----------|--------|
| 0.0 | 없음 |
| 0.1-3.9 | 낮음 |
| 4.0-6.9 | 중간 |
| 7.0-8.9 | 높음 |
| 9.0-10.0 | 치명적 |

### 활용과 한계

**활용:**
- 취약점 우선순위 지정
- 패치 관리 결정
- 보안 보고서 작성

**한계:**
- 실제 악용 가능성 미반영
- 조직별 맥락 미고려
- 점수만으로 위험 판단 위험
