## 정의

**동적 분석(Dynamic Analysis)**은 프로그램을 실제로 실행하면서 런타임 동작을 관찰하고 분석하는 기법이다.

## 핵심 개념

### 분석 방식
- **테스팅**: 다양한 입력으로 실행하여 오류 확인
- **퍼징**: 무작위/변형 입력으로 크래시 유발
- **계측(Instrumentation)**: 코드에 분석 코드 삽입
- **모니터링**: 메모리, 네트워크 등 실행 상태 감시

### 장점
- 실제 실행 동작 확인
- 런타임 취약점 발견 (메모리 누수, 경쟁 조건 등)
- 오탐률이 낮음

### 단점
- 코드 커버리지 한계
- 실행 환경 의존성
- 시간 비용이 큼

## 대표 도구

- **[[퍼징]]**: AFL, libFuzzer, OSS-Fuzz
- **메모리 검사**: Valgrind, AddressSanitizer
- **동적 계측**: DynamoRIO, Intel Pin

## 관련 개념

- [[정적 분석]]: 실행 없이 코드 분석
- [[CVE]]: 동적 분석으로 발견되는 취약점들
