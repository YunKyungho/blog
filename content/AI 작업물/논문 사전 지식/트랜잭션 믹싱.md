# 트랜잭션 믹싱 (Transaction Mixing)

**트랜잭션 믹싱**은 [[암호화폐]] 거래에서 발신자와 수신자 간의 연결을 끊어 프라이버시를 보호하는 기술이다.

## 핵심 개념

### 필요성
- [[블록체인]]의 모든 거래는 공개적으로 기록됨
- 주소 분석으로 사용자 신원 추적 가능
- 금융 프라이버시 보호 필요

### 기본 원리
1. 여러 사용자의 자금을 모음
2. 자금을 섞어서 원래 출처 파악 불가능하게 만듦
3. 각 사용자에게 동일한 금액 분배

## 주요 기술

### 중앙화 믹서 (Centralized Mixers)
- **작동 방식**: 신뢰할 수 있는 제3자가 자금을 수집하고 재분배
- **장점**: 구현이 간단
- **단점**: 신뢰 필요, 자금 도난 위험, 로그 보관 가능

### CoinJoin
- **작동 방식**: 여러 사용자가 하나의 트랜잭션에 입력/출력 결합
- **예시**: Wasabi Wallet, JoinMarket
- **장점**: 탈중앙화, 자금 보관 불필요
- **단점**: 참여자 조정 필요

### 영지식 증명 기반 (Zero-Knowledge Proof)
- **작동 방식**: 암호학적으로 거래 연결 끊음
- **예시**: Tornado Cash, Zcash
- **장점**: 강력한 프라이버시
- **단점**: 복잡한 암호학, 높은 가스 비용

### 링 서명 (Ring Signatures)
- **작동 방식**: 여러 가능한 서명자 중 누가 서명했는지 숨김
- **예시**: Monero
- **장점**: 프로토콜 수준 통합
- **단점**: 트랜잭션 크기 증가

## 규제 이슈

### 법적 논쟁
- 자금세탁 도구로 악용 가능성
- 2022년 Tornado Cash OFAC 제재
- 합법적 프라이버시 vs 불법 사용

### 규제 대응
- 일부 국가에서 믹서 사용 금지
- KYC(신원확인) 요구 증가
- 규정 준수 믹서 등장
