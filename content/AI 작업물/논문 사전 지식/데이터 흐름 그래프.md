## 정의

**데이터 흐름 그래프(Data Flow Graph, DFG)**는 프로그램에서 데이터가 어떻게 정의되고 사용되는지를 그래프로 표현한 것이다. 변수의 정의-사용 관계를 추적한다.

## 핵심 개념

### 핵심 분석
- **정의(Definition)**: 변수에 값이 할당되는 지점
- **사용(Use)**: 변수 값이 읽히는 지점
- **정의-사용 체인**: 정의에서 사용까지의 경로

### 활용 분야
- **컴파일러 최적화**: 상수 전파, 복사 전파
- **취약점 탐지**: 오염된 데이터(tainted data) 추적
- **슬라이싱**: 특정 변수에 영향을 주는 코드 추출
- **[[딥러닝]] 분석**: 데이터 의존성 학습

### 오염 분석 예시
```
소스(사용자 입력) → 변수 A → 함수 호출 → 변수 B → 싱크(SQL 쿼리)
```
→ SQL 인젝션 취약점 경로 탐지 가능
