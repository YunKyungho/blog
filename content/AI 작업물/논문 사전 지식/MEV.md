# MEV (Maximal Extractable Value)

**MEV(Maximal Extractable Value)**는 [[블록체인]] 네트워크에서 블록 생성자가 트랜잭션 순서를 조작하여 추출할 수 있는 최대 가치를 의미한다.

## 핵심 개념

### 정의의 변화
- **원래 명칭**: Miner Extractable Value (채굴자 추출 가치)
- **현재 명칭**: Maximal Extractable Value (이더리움 PoS 전환 후)
- 검증자, 시퀀서 등 블록 생성 권한을 가진 모든 주체에 적용

### 발생 원리
1. 사용자가 트랜잭션을 전송하면 **멤풀(Mempool)**에 대기
2. 블록 생성자가 트랜잭션 순서를 결정
3. 순서 조작으로 이익 추출 가능

### 주요 MEV 유형

#### 프론트러닝 (Front-running)
- 수익성 있는 거래를 발견하고 먼저 실행
- 예: 대량 구매 주문 앞에 미리 구매

#### 백러닝 (Back-running)
- 특정 거래 직후에 거래 실행
- 예: 가격 변동 후 차익거래

#### 샌드위치 공격 (Sandwich Attack)
- 피해자 거래 앞뒤로 자신의 거래 삽입
- 프론트러닝 + 백러닝의 결합

#### 청산 MEV
- DeFi 대출 프로토콜의 청산 기회 선점
- 담보 부족 포지션을 먼저 청산

## 영향

### 부정적 영향
- 일반 사용자에게 **숨겨진 세금** 역할
- 슬리피지 증가로 거래 비용 상승
- 네트워크 혼잡 및 가스비 상승
- 탈중앙화 원칙 훼손

### 긍정적(?) 측면
- 시장 효율성 향상 (차익거래)
- 청산 메커니즘 작동 지원

## 완화 전략

### 프로토콜 수준
- **Flashbots**: MEV 경매 시스템으로 투명화
- **암호화 멤풀**: 트랜잭션 내용 숨김
- **공정한 순서 결정**: 타임스탬프 기반 정렬

### 애플리케이션 수준
- 프라이빗 트랜잭션 풀
- 커밋-리빌 스킴
- 분산 오라클 사용
