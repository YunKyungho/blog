## 정의

온톨로지(Ontology)는 특정 도메인 내의 개념, 범주, 속성, 관계를 명시적이고 형식적으로 정의한 지식 표현 체계이다. 컴퓨터가 해당 도메인의 지식을 이해하고 추론할 수 있도록 한다.

## 상세 설명

### 온톨로지 구성요소

1. **클래스(Class)**: 개념의 범주 (예: 취약점, 소프트웨어)
2. **인스턴스(Instance)**: 클래스의 구체적 예시 (예: CVE-2024-1234)
3. **속성(Property)**: 개체의 특성 (예: CVSS 점수)
4. **관계(Relation)**: 개체 간 연결 (예: "영향을 미침")
5. **공리(Axiom)**: 도메인 규칙 및 제약조건

### 표현 언어

- **OWL (Web Ontology Language)**: W3C 표준
- **RDF (Resource Description Framework)**: 기본 그래프 표현
- **RDFS**: RDF 스키마
- **SPARQL**: 온톨로지 질의 언어

### 사이버보안 온톨로지 예시

**취약점 도메인:**
```
Vulnerability
  ├── hasID (CVE-2024-1234)
  ├── affectsProduct (CPE:...)
  ├── hasSeverity (Critical)
  └── isTypeOf (RemoteCodeExecution)
```

**활용 분야:**
- CVE 자동 분류 및 분석
- 위협 인텔리전스 통합
- 취약점-공격 패턴 매핑
- 보안 지식 추론

### LLM과의 결합

- 온톨로지로 LLM 출력 구조화
- 도메인 지식으로 LLM 성능 향상
- 환각 현상 감소
