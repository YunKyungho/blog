# 블록체인 상호운용성 (Blockchain Interoperability)

**블록체인 상호운용성**은 서로 다른 [[블록체인]] 네트워크 간에 데이터와 자산을 안전하게 교환할 수 있는 능력을 의미한다.

## 핵심 개념

### 필요성
- 수백 개의 독립적인 블록체인 존재
- 각 체인이 고립된 "섬"처럼 운영
- 자산과 데이터의 자유로운 이동 필요

### 주요 과제
1. **합의 차이**: 각 체인마다 다른 합의 메커니즘
2. **데이터 형식**: 호환되지 않는 트랜잭션 구조
3. **신뢰 모델**: 탈중앙화 수준의 차이
4. **보안 보장**: 크로스체인 공격 방지

## 상호운용성 접근법

### 1. 사이드체인 (Sidechains)
- 메인 체인에 연결된 별도의 블록체인
- 양방향 페그(peg)로 자산 이동
- 예: Liquid Network (비트코인)

### 2. 크로스체인 브릿지 (Cross-chain Bridges)
- 두 체인 간 자산 전송을 중개
- 락앤민트(Lock-and-Mint) 방식 사용
- 예: Wormhole, Multichain

### 3. 해시타임락 컨트랙트 (HTLC)
- 원자적 스왑(Atomic Swap) 구현
- 시간 제한과 해시 비밀을 이용
- 신뢰 없는 교환 가능

### 4. 릴레이 체인 (Relay Chains)
- 여러 체인을 연결하는 허브 역할
- 예: Polkadot, Cosmos IBC
- 공유 보안 모델 제공

### 5. 공증인 방식 (Notary Schemes)
- 신뢰할 수 있는 제3자가 거래 검증
- 중앙화되지만 구현이 간단
- 다중 서명으로 신뢰 분산

## 보안 고려사항

### 브릿지 해킹 사례
- **Ronin Bridge** (2022): 6억 달러 피해
- **Wormhole** (2022): 3억 달러 피해
- **Nomad** (2022): 1.9억 달러 피해

### 위험 요소
- 스마트 컨트랙트 취약점
- 검증자 담합
- 재진입 공격
- 오라클 조작
