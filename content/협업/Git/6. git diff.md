이번엔 변경 전 후의 차이를 볼 수 있는 git diff 명령어를 알아보자.

---

git diff를 입력하면 아래와 같은 출력 값이 나온다.

(안 나오면 diff가 없는 것이니 echo text >> fileName 명령어 등으로 아무 내용이나 추가하자.)

![](images/Git/23.png)

(직관적으로 알아보기 쉽게 파일내용을 추가, 삭제 해봤다.)

  

**diff --git a/test.txt b/test.txt**

어떤 파일에 대한 diff 인지를 나타낸다.

a는 변경 전 test.txt 파일을 의미하며 b는 변경 후 test.txt 파일을 의미한다.

  

**idnex (hash)**

git에서 내부적으로 파일내용을 참조할 때 쓰는 값이라고 한다.

  

**@@ -1,2 +1, 3 @@**

여러번 테스트 해봤는데 확실하게는 모르겠다.

-1(이전 파일)은 2줄의 내용이 있었고 +1(수정 뒤의 파일)은 3줄의 내용이 있다. 라는 의미로 추정된다.

내용이 1줄만 존재할 때는 -1이나 +1 뒤에 따로 숫자 표기는 되지 않는다.

  

그리고 이 뒤에 나오는 내용들은 -는 삭제된 내용 +는 추가된 내용 그냥 흰글 씨는 기존 내용을 의미한다.

![](images/Git/24.png)

위 사진을 보면 알 수 있듯이 파일 내용의 순서 그대로 차례대로 출력 된다.

  

근데 diff  같은 경우 굳이 눈알 빠지게 터미널로 보고 있을게 아니라 소스트리 같은 UI 툴로 보는게 훨씬 편할 것 같다.

![](images/Git/25.png)

아주 직관적이다.
