이번엔 .gitignore 파일에 대해 알아보자.

  

ignore 단어 뜻에서 알 수 있듯이 이 파일에 적힌 위치 혹은 유형의 파일은 무시해버린다.

---

프로젝트를 빌드하는 도중 생기는 log 파일 같은 부수적인 파일들을 git에 추가하고 싶지 않을 때 매번 add 단계에서 제외 시키거나 staging area에서 빼주는 과정이 귀찮을 때 사용하는 방식이다.

  

항상 github에서 이건 뭐지? 하면서 보던 .gitignore 파일의 정체를 드디어 알게되었다.

![](images/Git/20.png)

git status 확인 시 위 처럼 test1.log 파일은 제외 돼있는 것을 볼 수 있다.

  

![](images/Git/21.png)

위 같은 방식으로도 추가할 수 있다.

echo로 *.log를 추가 했을 때는 기존 파일의 파일명으로 바뀌어서 입력됬는데 이 방식은 아닌 것 같다.

어떤 특정 디렉토리 하위로도 지정이 가능하고 특정 디렉토리의 하위의 모든 log 파일로도 지정 가능하다.

  

![](images/Git/22.png)

그리고 역시 git status를 확인해보면 지정해둔 모든 파일들을 tracking 하지 않는 것을 볼 수 있다.
