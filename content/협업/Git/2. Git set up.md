이번엔 Git의 기본 설정 등을 알아보자.

---

**다운로드**

  

터미널을 열고 git --version을 입력하여 git이 설치가 돼있는지 확인한다.

  

없다면 [공식사이트](https://git-scm.com/)에서 다운 받는다.

  

git을 사용하는 방식은 2가지로 나뉜다.

1. 터미널로 직접 명령어를 입력하는 방식.

2. GUI가 있는 어플리케이션을 이용하는 방식.

  

git 명령어에 대한 전반적인 이해가 필요하기에 git을 공부하는 것이지만 회사에서 sublime merge라는 프로그램을 사용하고 있어서 나 같은 경우 1번과 2번 둘 다 익숙해지려고 한다.

  

GUI 프로그램은 **Atlassian**에서 만든 **Sourcetree**가 전반적으로 많이 쓰인다고 하여 이용해보려고 한다.

없다면 [공식사이트](https://www.sourcetreeapp.com/)에서 다운 받는다.

  

어떤 툴을 쓰더라도 기존 커맨드의 일부로 구성되어 있기에 git을 처음 사용하거나 git에 정말 익숙해지고 싶다면 터미널을 이용하는 것이 좋다고 한다.

  

터미널은 OS에 내장된 기본 터미널도 괜찮지만 맥은 [iTerm2](https://iterm2.com/) 윈도우는 [cmder](https://cmder.app/)을 사용하는 것도 좋다.

**기본 설정**

  

git config --list

라는 명령어로 모든 설정을 확인 가능하다.

  

git config --global -e

라는 명령어로 설정 편집이 가능하다.

  

만약 터미널에서의 텍스트 편집이 어렵고 VSCode가 설치되어있다면

  

git config --global core.editor "code"

git config --global -e

를 통하여 에디터를 지정하고 VSCode로 설정 편집이 가능하다. (VSCode 터미널 명령어 설정이 돼있어야된다.)

  

git config --global user.name "UserName"

git config --global user.email "email@email.com"

으로 user 이름과 이메일 설정도 가능하다.

  

git config user.name

git config user.email

로 설정한 이름과 이메일 확인이 가능하다.

  

git config --global core.autocrlf true(window) | input(mac)

운영체제에 따라 명령어를 잘 입력해준다.

운영체제별로 엔터키 정책이 달라 \r\n으로 인한 git 저장소, git history가 꼬이는걸 방지해준다고 한다.

**명령어**

  

Git 명령어는 [공식문서](https://git-scm.com/doc)에서 전부 확인 가능하다.

모든 명령어를 달달 외우기 보단 자주 쓰는 명령어에 익숙해지는 것이 먼저다.

  

본인의 프로젝트 최상위 폴더로 이동한 뒤

git init

명령어를 입력 시 git이 초기화 되며 .git이라는 숨김 폴더가 생성된다. 이 안에서 git에 대한 모든 정보를 확인할 수 있다.

  

--global은 config 명령어의 옵션이다. 이런 옵션들을 한번에 확인하고 싶다면

git config --h

와 같이 git 명령어 --h로 해당 명령어의 모든 옵션을 간단하게 확인할 수 있다.

---

**단축키 만드는 법**

  

자주쓰는 명령어가 너무 길어 불편하다면 아래 형식의 명령어로 단축키를 만들 수 있다.

  

git config --global alias.st status

git status 명령어를 git st로 줄여놓은 것이다.
