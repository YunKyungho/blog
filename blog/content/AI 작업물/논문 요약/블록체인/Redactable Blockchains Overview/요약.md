## 한줄 요약

불변성이 핵심인 블록체인에서 규제 준수, 오류 수정, 민감정보 삭제 등을 위해 데이터를 수정할 수 있게 하는 수정 가능 블록체인 기술을 개관한다.

## 핵심 내용

[[블록체인]]의 **불변성(Immutability)**은 데이터 무결성과 투명성을 보장하지만, 현실에서는 수정이 필요한 상황이 존재한다:

### 수정 필요 상황

1. **규제 준수**: GDPR의 '잊힐 권리' 등
2. **오류 수정**: 잘못 기록된 데이터 정정
3. **민감정보 제거**: 개인정보 보호
4. **불법 콘텐츠 삭제**: 법적 의무

### 주요 접근법

1. **카멜레온 해시**: 특정 조건에서 해시 충돌 생성 가능
2. **편집 가능 합의**: [[합의 알고리즘]] 수준에서 수정 허용
3. **오프체인 저장**: 민감 데이터를 체인 외부에 보관
4. **투표 기반 수정**: 참여자 동의로 수정 승인

### 트레이드오프

- 보안성 vs 유연성
- 검열 저항성 vs 규제 준수
- 불변성 신뢰 vs 수정 가능성

## 주요 개념

- [[블록체인]]: 분산 원장 기술 (전통적으로 불변)

- [[스마트 컨트랙트]]: 수정 정책 자동화에 활용 가능
